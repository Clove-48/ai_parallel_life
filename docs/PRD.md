## 1. 产品概述

### 1.1 产品定位

**AI 平行人生** 是一款基于 AI 叙事生成的情感体验类 Web 产品。用户描述人生关键节点，AI 推演"如果当初做了不同选择"的平行叙事，以沉浸式多媒介形式呈现，帮助用户在"真实线"与"平行线"的对比中获得情感共鸣与自我认知。

### 1.2 产品愿景

让每个人都能与自己的过去对话。AI 不是答案，而是镜子。

### 1.3 核心理念

> 每一条路都有风景，重要的不是选哪条路，而是认真走好选择的路，找到当下的意义。

### 1.4 产品 slogan

**What If You Had Chosen Differently?**

### 1.5 竞品差异化

| 维度 | 常规工具型产品 | AI 平行人生 |
|------|---------------|-------------|
| 产品类型 | 工具型（效率、实用、功能驱动） | **体验型（情感叙事、沉浸式体验）** |
| AI 角色 | 辅助者、效率工具 | **叙事构建者、情感共鸣者** |
| 用户情感 | "这东西有用" | **"这东西触动了我"** |
| 传播力 | 依赖实用价值 | **依赖情感共鸣，天然分享欲** |
| 记忆点 | 功能列表 | **一个故事、一次体验** |
| 竞争壁垒 | 功能可复制 | **情感体验难以复制** |

## 2. 目标用户

### 2.1 用户画像

| 画像 | 年龄 | 核心场景 | 情感驱动 |
|------|------|----------|----------|
| 人生过渡期人群 | 22-35 岁 | 毕业、换工作、分手、搬家——选择密集期 | 怀旧、好奇、自我确认 |
| 情感内省型用户 | 25-40 岁 | 写日记、回顾人生、自我探索 | 反思、寻求意义感 |
| 内容创作者 | 25-40 岁 | 寻找有情感深度的社交内容素材 | 创作驱动、社交货币 |
| 决策焦虑人群 | 22-35 岁 | 面对重大选择时反复纠结 | 情感疗愈、告别仪式 |

### 2.2 用户场景示例

- **深夜独处场景**: 用户躺在床上，回想"如果当初去了另一个城市"，打开产品体验
- **职业倦怠场景**: 上班族在通勤路上想象"如果当初选择了另一条路"
- **情感疗愈场景**: 刚经历分手，想"看看如果当初没分开"
- **社交分享场景**: 内容创作者生成感悟卡，截图发朋友圈引发讨论

## 3. 用户旅程

### 3.1 核心流程

```
[进入] → [对话式回忆] → [AI 生成叙事] → [沉浸式翻阅] → [感悟卡片] → [分享/离开]
```

### 3.2 分步描述

| 步骤 | 页面/状态 | 用户行为 | 系统响应 | 情感目标 |
|------|-----------|----------|----------|----------|
| 1. 进入 | 首页 | 打开产品，看到引导语 | 展示温暖简洁的欢迎界面 | 安全、私密 |
| 2. 回忆 | 对话界面 | 描述关键人生节点 | AI 逐步提问引导（时间、地点、选择、结果） | 被倾听、被引导 |
| 3. 生成 | 加载中 | 等待 AI 生成叙事 | 显示生成进度，氛围过渡动画 | 期待、好奇 |
| 4. 翻阅 | 时间线界面 | 滑动浏览两条叙事线 | 以多媒介卡片形式呈现内容 | 沉浸、触动 |
| 5. 感悟 | 结果页 | 查看感悟卡 | AI 提炼情感线索，生成卡片 | 释然、共鸣 |
| 6. 分享 | 分享弹窗 | 保存/分享感悟卡 | 提供 Canvas 生成图片、复制链接 | 满足、连接 |

## 4. 功能需求

### 4.1 功能清单

| 编号 | 功能模块 | 优先级 | 描述 |
|------|----------|--------|------|
| F1 | 欢迎引导页 | P0 | 首屏展示品牌理念 + 引导语 + 开始按钮 |
| F2 | 对话式节点采集 | P0 | AI 以提问引导用户描述人生关键节点 |
| F3 | 双线叙事生成 | P0 | AI 生成"真实线"+"平行线"叙事 |
| F4 | 沉浸式时间线展示 | P0 | 以多媒介卡片形式展示叙事内容 |
| F5 | 感悟卡片生成 | P0 | 体验结束后生成精美图文卡片 |
| F6 | 分享功能 | P0 | 支持截图保存、复制链接、分享到社交平台 |
| F7 | 历史记录 | P1 | 用户可查看和管理过往的平行人生故事 |
| F8 | 加载/过渡动画 | P1 | 叙事生成过程中的氛围动画 |

### 4.2 功能详情

#### F1: 欢迎引导页

- 展示品牌名称、slogan、核心理念
- 一句引导语："每个人心中都有一个'如果当初'。你想回到哪个时刻？"
- 一个显著的 CTA 按钮："开始体验"
- 背景氛围：温暖、柔和、私密

#### F2: 对话式节点采集

- AI 作为引导者，逐步提问：
  1. "那是什么时候？"（时间）
  2. "当时你在哪里？"（地点）
  3. "你面临的选择是什么？"（选择 A vs 选择 B）
  4. "你最终选择了什么？"（真实选择）
  5. "结果怎么样？"（真实结果）
  6. "如果做了另一个选择，你觉得会怎样？"（用户的想象引导）
- 界面模拟即时通讯的对话风格
- 用户打字输入，AI 逐条回复
- 支持自由输入，不限于固定格式

#### F3: 双线叙事生成

- AI 基于用户输入生成两条并行叙事线：
  - **真实线**: 对用户真实经历的情感化回溯（温暖、接纳）
  - **平行线**: "如果当初做了不同选择"的推演（想象力、可能性）
- 每条线包含 3-5 个关键场景/片段
- 叙事风格：文学化、情感化、第一/第二人称混合

#### F4: 沉浸式时间线展示

- 界面设计为上下分屏或左右分屏：
  - 左侧/上方：真实线（暖色调）
  - 右侧/下方：平行线（梦幻色调）
- 每条线的场景以卡片形式呈现，卡片类型包括：
  - **聊天记录卡片**: 模拟微信/iMessage 对话截图风格
  - **朋友圈卡片**: 模拟朋友圈时间线风格
  - **照片卡片**: AI 生成的场景描述 + 占位图或 SVG 插画
  - **语音日记卡片**: 模拟语音消息界面
  - **日记条目卡片**: 模拟手写日记风格
- 用户通过滚动/滑动翻阅，卡片有渐入动画

#### F5: 感悟卡片生成

- AI 分析两条叙事线，提炼共同情感线索
- 生成一张精美图文卡片，包含：
  - 一句话核心感悟
  - 两条叙事线的象征性对比
  - 温暖鼓励性的结语
- 纯前端 Canvas 或 CSS 渲染（无需后端图片处理）
- 支持自定义背景色/主题

#### F6: 分享功能

- 分享方式：
  - 截图保存（通过 `html2canvas` 或 Canvas 渲染）
  - 复制分享文案
- 分享内容：感悟卡片 + 产品信息
- 无统计追踪要求

#### F7: 历史记录

- 列表展示用户过往的平行人生故事
- 每条记录显示：标题、日期、摘要
- 支持点击重新查看完整叙事
- 支持删除单条记录

### 4.3 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| N1 | 首屏加载时间 | < 3s |
| N2 | AI 生成响应时间 | < 15s（含流式输出） |
| N3 | 移动端适配 | 完美适配 320px - 768px 宽度 |
| N4 | 离线能力 | 历史记录支持本地缓存 |
| N5 | 隐私保护 | 用户数据加密存储，不用于模型训练 |

## 5. UI/UX 需求

### 5.1 设计风格

| 维度 | 要求 |
|------|------|
| 风格 | 温暖、柔和、文艺、私密 |
| 配色 | 暖棕色系（#3a3226 / #c8842c）+ 紫色点缀（#8b6faa） |
| 字体 | 中文字体：系统无衬线；英文衬线字体用作标题装饰 |
| 氛围 | 低饱和度背景，圆角卡片，柔和阴影 |
| 动效 | 克制使用，以渐入和淡出为主，不喧宾夺主 |

### 5.2 页面结构

1. **首页（欢迎页）**
   - Hero 区域：标题 + 引导语 + CTA
   - 可选的：几张示例卡片轮播

2. **对话采集页**
   - 类聊天界面
   - AI 消息在左，用户消息在右
   - 输入框在底部

3. **加载/生成页**
   - 过渡动画
   - 提示文字（如"正在翻阅另一个时空的你……"）

4. **时间线展示页**
   - 两条并行的时间线
   - 卡片从上到下排列
   - 滑到末尾触发感悟卡生成

5. **感悟卡 + 分享页**
   - 展示最终感悟卡
   - 分享按钮组

### 5.3 响应式断点

| 断点 | 布局 |
|------|------|
| ≥ 768px | 桌面布局，双列时间线 |
| < 768px | 移动布局，上下切换时间线 |
| < 480px | 小屏优化，字体和间距缩小 |

### 6.1 数据结构

**用户故事记录**
```json
{
  "id": "uuid",
  "createdAt": "iso-datetime",
  "title": "如果当初去了北京",
  "node": {
    "time": "2019年夏天",
    "location": "成都",
    "choiceA": "留在成都",
    "choiceB": "去北京",
    "actualChoice": "留在成都",
    "actualOutcome": "在成都稳定工作至今",
    "imagination": "可能会在互联网公司打拼"
  },
  "narratives": {
    "real": [
      { "type": "chat", "content": "...", "scene": "..." },
      { "type": "photo", "content": "...", "scene": "..." },
      { "type": "diary", "content": "...", "scene": "..." }
    ],
    "parallel": [
      { "type": "chat", "content": "...", "scene": "..." },
      { "type": "moment", "content": "...", "scene": "..." },
      { "type": "voicenote", "content": "...", "scene": "..." }

  },
  "reflectionCard": {
    "insight": "你一直渴望的，不是去远方，而是选择的勇气",
    "themeColor": "#c8842c"
  }

```

### 6.2 存储方案

- **后端数据库**: SQLite（轻量，适合单机/小规模部署）
- **前端缓存**: localStorage（历史记录快速读取）
- **图片资源**: 无用户上传图片，所有"照片"为 AI 文字描述 + 前端占位插画

## 7. 验收标准

### 7.1 功能性验收

| 编号 | 验收项 | 通过标准 |
|------|--------|----------|
| A1 | 完整流程闭环 | 用户从首页进入，完成回忆→生成→翻阅→分享全流程 |
| A2 | AI 叙事质量 | 生成的叙事具有情感温度，不空洞，不模板化 |
| A3 | 双线展示 | 真实线和平行线在界面上清晰区分，风格统一 |
| A4 | 感悟卡片 | 卡片美观、文案有共鸣，可截图保存 |
| A5 | 历史记录 | 用户可查看历史故事，数据持久化 |

### 7.2 体验性验收

| 编号 | 验收项 | 通过标准 |
|------|--------|----------|
| B1 | 加载体验 | 生成过程中有过渡动画，等待不焦虑 |
| B2 | 移动端手感 | 滑动流畅，触控反应灵敏 |
| B3 | 视觉一致性 | 所有页面风格统一，色调一致 |
| B4 | 文字可读性 | 字体大小、行距舒适，阅读无障碍 |
